# Review DB Schema

ローカルの SQLite データベース (`local-ai-review.db`) は、PRレビューの証跡、精度評価、および「Local Data Gravity」を活かした Bug Immune Network（学習ループ）の中核となるデータストアです。

## テーブル概要

### 実行履歴とメタデータ
- **`review_runs`**: レビュー実行ごとのメタデータ。PR情報、モデル、処理時間、プロンプトのハッシュなどを保存します。
- **`reviewed_files`**: 各レビュー実行で実際にモデルに送られたファイルのパス。
- **`artifacts`**: レビュー実行中に生成された追加ファイル（レポートやコンテキスト）のパスとSHA256ダイジェスト。
- **`runtime_metrics`**: パフォーマンス分析のための実行時の負荷やキュー、Ollamaのステータス情報。

### レビュー結果とフィードバック
- **`findings`**: ローカルLLMが発見した具体的な指摘事項。
- **`watch_items`**: 指摘には至らないが注視すべき（Watch）項目。
- **`run_feedback`**: 実行単位のユーザースコア（有用な指摘の数、誤検知の数など）。

### 統合アイテムスキーマ (v1.0 Evidence Loop)
従来の `findings` や `watch_items` から一歩進み、より精緻な学習ループを構築するためのテーブル群です。
- **`review_items`**: ローカルのレビューで発見されたすべてのアイテム。変更箇所の fingerprint や actionability が含まれます。
- **`external_items`**: GitHub の PR コメント等からインポートされた外部のレビュー・フィードバック。
- **`item_links`**: ローカルの `review_items` と外部の `external_items` を照合・リンクする関係情報。
- **`item_verdicts`**: 特定のアイテムに対する人間の判定結果（誤検知、有用、スコアなど）。

### 学習と継続的改善 (Learning)
- **`github_backfill_queue`**: 過去のGitHub PRを取り込み、学習用の外部アイテムとして収集するためのバックグラウンドキュー。
- **`rule_updates`**: レビュー結果や判定に基づいて生成・適用されたルールの変更履歴。
- **`learning_calibrations`**: 現在アクティブな学習データやプロンプトへのパッチ（キャリブレーション）。

## プライバシーとローカル・オンリー方針

このデータベースは「**自社コードを外に出さない**」という原則の元で動作しています。

### プライバシー保護フィールド
`review_items` や `external_items` には、`body` (コメント本文) や `fix` (修正案) などの生のテキストが含まれますが、これらは **ローカルマシン上にのみ保持** されます。学習用のエクスポートを行う場合は、`diff_fingerprint` などのダイジェスト情報や、統計的な `path_class` 情報だけが共有され、生のコードスニペットは外部に送信されません。

### エクスポート安全ポリシー (Export-safe)
商用の immune network（バグ免疫ネットワーク）へ学習フィードバックを提供する場合、エクスポート対象となるのは `item_verdicts`（判定のラベル）や、ハッシュ化された `fingerprint` に限られます。これにより、ソースコードの機密性を保ちながら学習ループに参加することが可能です。

## マイグレーションポリシー

データベースのスキーマ変更は、`scripts/local-ai-precision-review.py` の `migrate_db_schema` 関数で行われます。
- 新しいカラムの追加は `ALTER TABLE ... ADD COLUMN` で安全に処理されます。
- ビュー (`review_run_summary`) はスキーマ更新時に自動的に再作成されます。
- 大きな破壊的変更は行わず、前方互換性を保ちながら追加のテーブルやカラムで対応します。
